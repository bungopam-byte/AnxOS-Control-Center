# Example AGENTS.md

Use this template when starting another AnxOS-related repository.

## Rules

- Inspect existing code and docs before changing behavior.
- Preserve the product's existing visual identity.
- Keep security-sensitive functionality protected in trusted backend or main-process code.
- Never expose secrets in renderer code, logs, diagnostics, or committed files.
- Run relevant validation and report exact command results.
- Use Conventional Commit messages.
- Preserve unrelated changes and avoid destructive Git commands.
