Local Agent

Run Everything on Your Own PC

The AnxOS Local Agent securely connects the desktop app to services running on your computer. It handles server processes, files, dependencies, backups, and networking while keeping your data on your own machine.

Normal Windows users do not need Anx's Debian server. The Local Agent can run as a Windows service, remote servers remain optional, authentication is configured automatically, and you can stop or uninstall it when you no longer need it.

Runs Locally

Manage servers and files on This PC without sending work to a developer-owned Debian machine.

Automatic Setup

AnxOS installs the managed Agent runtime and prepares the folders it needs from inside the app.

Starts with Windows

The Local Agent can run as a Windows service, so the desktop app does not need to stay open.

No Token Copying

Local pairing is handled automatically on your own computer without asking you to paste Agent tokens.

Remote Servers Optional

You can still add a VPS, NAS, Debian server, Windows Server, or Raspberry Pi later.

Easy Repair and Updates

Agent diagnostics, repair, and updates stay inside AnxOS instead of requiring terminal commands.

Getting Started

First server workflow

These steps match the current desktop workflow. AnxOS does not install unsupported runtimes or approve remote access without your action.

1

Download AnxOS Control Center for Windows or Linux.

2

Install and launch the desktop application.

3

Start the local AnxOS Agent or connect a remote Agent.

4

Add or select the Node you want to manage.

5

Open Prepare Node to verify Java, Docker, SteamCMD, Node.js, Python, and other required runtimes.

6

Install your first Marketplace server after dependency checks pass.

7

Monitor the server through Operations, Diagnostics, Files, and Node Health.

Install Guide

Install in a few steps

1

Download the installer for your platform.

2

Run the Windows setup, install the `.deb`, or mark the AppImage executable.

3

Open AnxOS, connect your remote node, then use Marketplace, Instances, Files, Console, and Stats.
