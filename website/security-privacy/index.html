Local Agent safety

Security and Privacy

The Local Agent runs on your own computer. It connects AnxOS Control Center to local server processes, files, dependencies, backups, and networking tools selected inside the app.

Runs on your PC

The Local Agent is installed on the Windows computer you choose. Instances, backups, logs, and downloaded server files remain in the configured local storage locations unless you intentionally use a remote node or provider.

Requested actions

The Agent is intended to perform actions requested through AnxOS, such as starting a server, editing an instance file, creating a backup, or installing a selected dependency.

Automatic authentication

Local pairing is configured automatically for this computer. Normal users should not need to copy Agent tokens, edit configuration files, or paste secrets into the app.

Remote access

Remote access is not enabled by default. Playit, Tailscale, Cloudflare Tunnel, and manual port forwarding are optional choices that may require provider accounts, firewall rules, or router changes.

Diagnostics

Logs and diagnostics are designed to sanitize secrets before display or export. Full tokens, API keys, and private environment values should not be shown in normal diagnostics.

Control

You can stop the Local Agent service or uninstall the Agent. Removing the Agent should not remove your user-created instances or backups unless an uninstall option explicitly says it will.

Important warning

Server software can change your exposure

Installing server software can open local services, expose ports, create firewall prompts, or download third-party software depending on the templates, dependencies, and public access providers you choose. Review each install prompt before approving it.
